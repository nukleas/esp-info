# ESP32 Reference Website - Claude Code Prompts

This collection of prompts is designed to help you build a comprehensive, interactive ESP32 reference website using Claude Code (or Claude.ai).

## What This Project Is

An interactive web application that consolidates ESP32 documentation, pinout diagrams, variant comparisons, and common wiring examples into one easy-to-use resource. Think of it as a developer-friendly quick reference for working with ESP32 chips.

- üìå **Interactive Pinout Diagrams** - Visual, clickable chip diagrams with detailed pin information

- üîå **Common Circuits** - Library of frequently used wiring examples with schematics

## How to Use These Prompts

### Option 1: Claude Code (Recommended)

1. Install [Claude Code](https://claude.ai/code) if you haven't already
2. Start with `00-implementation-guide.md` to understand the full scope
3. Follow the "Recommended Prompt Sequence" in that file
4. Copy each session's prompt and paste it into Claude Code
5. Review the generated code and iterate as needed

### Option 2: Claude.ai with Computer Use

1. Enable "Computer Use" in your Claude.ai conversation
2. Upload all markdown files to the conversation
3. Ask Claude to read `00-implementation-guide.md` and start with Session 1
4. Work through each session, letting Claude create files locally
5. Copy the generated project to your GitHub repo

## Technology Stack

- **Framework**: React 18+
- **Build Tool**: Vite
- **Styling**: Tailwind CSS
- **Routing**: React Router
- **Graphics**: SVG (for diagrams)
- **Fonts**: JetBrains Mono, Inter (from Google Fonts)

No complex dependencies - keeps it lightweight and fast!

## Does This Need MCPs?

**Short answer: No**

The project can be built entirely without MCPs. However, these are helpful:

- **Web Search MCP** - For gathering accurate ESP32 specs during development
- **GitHub MCP** - For automated commits (though you can do this manually)

Everything else (UI, components, logic) can be generated by Claude Code natively.

## What You'll Need to Provide

Claude Code can generate the entire structure and UI, but you'll need to provide:

1. **Actual ESP32 Specifications**
   - Pin functions for each variant
   - Technical specs (CPU, memory, etc.)
   - These can be found on [Espressif's official site](https://www.espressif.com/)

2. **Circuit Schematics** (optional)
   - Can be created as SVG by Claude
   - Or you can provide images/diagrams

3. **Testing & Iteration**
   - Review generated code
   - Test on different devices
   - Provide feedback for improvements

## Design Philosophy

This project emphasizes:

- **Performance** - No heavy animations, GPU-accelerated only
- **Usability** - Clean, intuitive interface
- **Accessibility** - Keyboard nav, screen reader support
- **Mobile-First** - Works great on phones
- **Developer-Friendly** - Built by devs, for devs
